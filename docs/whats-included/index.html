What's included?

1. Components

Common UI components that you'll need for your interface.

  • Components are CSS/HTML and they don't depend on any Javascript framework, so you can use them anywhere.
  • Components are made with Tailwind CSS but the output is a CSS file, so you can use the CSS file from CDN even if you're not using Tailwind CSS on your project.
  • You can see the list of all components here on this site.
Every component's CSS source code is separated in 2 files: base and styled. This allows you to skip using pre-designed components and use your own design if you want. See config document for more info.
  • `base` is the skeleton of components, no colors, no design decisions included.
  • `styled` includes all the colors, spacing and design decisions

2. Utility Classes

DaisyUI adds a few extra utility classes to Tailwind CSS. They are being used inside components.
You can see the available utilities on github

3. Resets

DaisyUI adds a tiny style reset to Tailwind's @base reset style.
You can see it on github

4. Themes optional

You can see all available themes on this site.

  • Default themes are available on CDN.
  • You can enable it from config file.
  • You can see the source for each theme on github.
  • You can add your own theme if you want.

5. full.css file available on CDN

This file includes a generated CSS including Tailwind CSS and DaisyUI.

  • It's useful if you want to quickly use Tailwind CSS and DaisyUI without installing anything.
  • It's not recommended to use this file on production because it's a large CSS file and nothing is purged.
  • full.css is available on CDN
